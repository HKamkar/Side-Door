# Mock interview loop

How to run an interactive mock. Don't simulate a whole panel — pick 4–6 questions, one at a time, give real feedback.

## Setup

- Confirm with the user: "Mock as {role}, interviewer voice as {recruiter / hiring manager / technical / panel}. 5 questions, ~20 minutes. Ready?"
- Pick 5 questions from the prep brief: 2 behavioral, 2 technical, 1 curveball.

## Per-question loop

1. **Ask** in the interviewer's voice. Match the style of the actual interviewer when you have profile data on them (e.g., terse, blunt, story-led, framework-led).
2. **Wait for the user's full answer.** Don't interrupt with side questions while they're answering.
3. **Give feedback** in exactly this shape:
   - **One thing that worked.** Specific — name the bullet or framing that landed.
   - **One thing to improve.** Specific — the moment you'd rewind. Length / structure / specificity / vocabulary.
   - **One suggested rephrasing.** Take ONE sentence the user said and rewrite it tighter or more concrete. Show, don't lecture.
4. **Offer:** "Want to redo this one, or move on?"

## Feedback rules

- **Don't be a yes-man.** If the answer was weak, say so. Use the brand voice the user set up in profile.
- **Don't be cruel.** The user is practicing. Direct, not harsh.
- **Don't grade out of 10.** Numerical grades feel fake. Use words: solid / serviceable / soft / weak.
- **Length matters.** Flag answers over 90 seconds (rough rule: more than ~200 words spoken) — interviewer attention drops fast.
- **Watch for "we" vs "I".** In behavioral answers, "we did X" is a red flag. Push for "I".
- **Watch for vague verbs.** "Helped with", "worked on", "was involved in" — push for "designed", "shipped", "owned", "decided".

## End of mock

Tally:
- 1–2 questions the user nailed — write down which ones, so they can reuse the framings.
- 1–2 questions to redo before the real interview — note why.
- Update the prep brief with these notes.

Ask if they want a second round on the weak ones, or to stop.

## Hard rules

- Don't coach the user to lie about gaps.
- Don't help the user prepare answers that misrepresent their experience.
- Don't simulate "trick" questions that interviewers ethically shouldn't ask (age, family, citizenship beyond authorization, religion). If the real interviewer asks one, your job is to help the user deflect respectfully, not to rehearse one.
